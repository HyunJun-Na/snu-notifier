# SNU 공지 알림 시스템

서울대 게시판(기계공학부 취업·장학·학부공지, SNU캘린더, 학생처)을 매일 아침 1회 확인해서
나에게 관련 있는 새 공지를 아침 리포트 한 통으로 정리해 텔레그램으로 보내주는 시스템.

- **1단계 규칙 필터**: 키워드로 확실한 것만 통과/제외
- **2단계 Claude 판정**: 애매한 글은 내 프로필 기준으로 0~10점 채점
- **3단계 라우팅**: 아침 리포트에서 🚨중요(8점↑) / ☀️일반(5~7점) 섹션으로 구분, 그 외 버림(목록 첨부)
- **프로필 실시간 참조**: https://hyunjun-na.github.io 페이지를 매 실행마다 읽어 판정에 반영
- **텔레그램 채팅으로 키워드 수정**: 폰에서 `/add`, `/block` 한 줄이면 끝

---

## 설치 (약 15분, 최초 1회)

### 1. 텔레그램 봇 만들기 (5분)
1. 텔레그램에서 **@BotFather** 검색 → `/newbot` → 이름 지정
2. 발급되는 **봇 토큰** 복사 (예: `1234567:ABC-DEF...`)
3. 만들어진 봇과의 채팅방에서 아무 메시지나 하나 보내기
4. 브라우저에서 `https://api.telegram.org/bot<봇토큰>/getUpdates` 접속
   → `"chat":{"id": 123456789` 에서 **chat id** 복사

### 2. Anthropic API 키 (3분)
1. https://console.anthropic.com → API Keys → 키 생성
2. 사용량은 Haiku 기준 월 몇백 원 수준 (최소 크레딧 $5 충전이면 1년 이상)

### 3. GitHub 저장소 (7분)
1. github.com에서 새 저장소 생성
   - **Public 권장** (Actions 무료 무제한. Private는 월 2,000분 제한이라 빠듯함)
   - 프로필·키워드가 공개되는 게 싫으면 Private + 실행 주기를 1시간으로 조정
2. 이 폴더의 파일 전체를 업로드 (`.github/workflows/monitor.yml` 경로 유지 주의!)
3. 저장소 **Settings → Secrets and variables → Actions** 에서 시크릿 3개 등록:
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CHAT_ID`
   - `ANTHROPIC_API_KEY`
4. **Actions 탭 → SNU Notice Monitor → Run workflow** 로 첫 실행
   → "초기 설정 완료" 메시지가 텔레그램으로 오면 성공.
   첫 실행은 기존 글을 기준선으로 저장만 하고, 이후부터 새 글만 알림.

## 텔레그램 명령어

| 명령 | 동작 |
|---|---|
| `/keywords` | 현재 통과/제외 키워드 전체 보기 |
| `/add 키워드` | 무조건 통과 목록에 추가 |
| `/block 키워드` | 무조건 제외 목록에 추가 |
| `/remove 키워드` | 목록에서 삭제 |
| `/profile` | 프로필 + 참조 페이지 확인 |
| `/status` | 마지막 실행 요약 |

명령은 **다음날 아침 실행 때 처리**됩니다 (봇이 확인 답장을 보냄). 바로 반영하고 싶으면
GitHub Actions 탭에서 "Run workflow" 버튼으로 수동 실행하면 즉시 처리됩니다.
변경사항은 config.yaml에 자동 커밋되므로 이력이 남고 언제든 되돌릴 수 있습니다.

## 프로필 수정 방법 (두 가지)
1. **github.io 페이지 수정** → 다음 실행부터 자동 반영 (권장)
2. config.yaml의 `profile.base` 직접 수정 → 커밋

## 운영 팁
- 첫 2~3주는 아침 리포트 하단의 "걸러진 글 목록"을 확인하고,
  필요한 게 걸러졌으면 `/add`로 조정하세요. 이후엔 거의 손댈 일 없음.
- 게시판이 추가로 필요하면 config.yaml `sources`에 URL 한 줄 추가.
- 알림이 안 오면: Actions 탭에서 실행 로그 확인 → `/status`로 오류 확인.
- GitHub은 60일간 저장소에 활동이 없으면 스케줄 실행을 멈춥니다.
  이 시스템은 상태 파일을 매번 커밋하므로 해당 없음.
